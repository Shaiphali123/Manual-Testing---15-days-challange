Day 10 - Test Execution

Test execution is the process of running test cases on the application to verify whether it meets the expected requirements. It is a crucial phase in manual testing to identify defects and ensure software quality.

Test Execution Process

1. Preparation

Ensure the test environment, test data, and all other prerequisites are ready before starting the execution.

2. Execution of Test Cases

Follow the documented steps in each test case meticulously to check the application's behavior.

3. Logging Results

Record the actual results observed during execution and compare them against the expected results.

4. Defect Reporting

Report any discrepancies, failures, or defects found with detailed steps, screenshots, and logs.

5. Re-Execution / Regression

After a defect is fixed, re-run the failed test cases to verify the resolution and related regression tests.

Test Execution Documentation

Proper documentation helps track testing progress, provides evidence of testing, and assists in future test planning. Key documents include:

  • Test Execution Report: Tracks the status of each test case (Pass, Fail, Blocked, etc.).
  • Defect Log: A detailed list of all defects found, including their status, severity, and priority.
  • Test Summary Report: A high-level overview of the entire testing effort, outcomes, and quality assessment.

Result Analysis

Analyzing the results of test execution is critical for understanding the state of the application. Key activities include:

  • Compare expected vs. actual results: The core of testing, identifying where the software deviates from requirements.
  • Determine Pass/Fail status: Assign a clear status to each test case based on the outcome.
  • Analyze defects and their severity/priority: Understand the impact of each defect to help prioritize fixes.
  • Summarize overall testing coverage and quality: Provide stakeholders with a clear picture of how much was tested and the software's stability.
