Day 7 - Bug & Defect Lifecycle

Introduction

In the world of software development, nothing is perfect. Bugs and defects are an inevitable part of the process. Understanding how to manage them is key to delivering high-quality software. A defect is a deviation from the expected behavior of software, and managing these defects efficiently is crucial for quality delivery. The Bug Lifecycle is a systematic process that tracks a bug from the moment it's found until it's resolved and verified.

Bug Lifecycle Stages

The bug lifecycle is a defined set of stages that a bug goes through from detection to closure. While the names might vary slightly, the core process remains the same across most organizations.

New
→
Assigned
→
Open
→
Fixed
→
Retest
→
Closed / Reopened

Note: The exact flow can vary slightly between different organizations and their project management tools.

  • New: A bug is first logged by a tester and is awaiting review by a project manager or team lead.
  • Assigned: The bug is assigned to a specific developer or development team to be fixed.
  • Open: The developer has started analyzing the bug and is working on a solution.
  • Fixed: The developer has completed the fix and marks the bug as ready for retesting by the QA team.
  • Retest: The QA team retests the bug fix to ensure the original issue is resolved and no new issues have been introduced.
  • Closed: If the bug is successfully verified as fixed, it is marked as closed.
  • Reopened: If the bug still persists after retesting, the tester reopens it and sends it back to the developer for further attention.

Severity vs Priority

When a bug is found, it's important to categorize it based on two key attributes: severity and priority.

  • Severity: The impact of the bug on the system or application. It measures "how bad" the bug is.
  • Priority: The urgency of fixing the bug. It determines "how soon" the bug needs to be fixed.
Severity Priority Description
Critical High The system crashes or a major feature is completely unusable. Must be fixed immediately.
Major Medium Significant functionality is impacted, but the system is still usable. Fix is required soon.
Minor Low A cosmetic issue or a very minor bug that doesn't impact core functionality. Can be fixed in later releases.

Interview Questions - Day 7

Q: What is a bug in software testing?

A: A bug is a defect, error, or fault in a software program that causes it to produce an incorrect or unexpected result.

Q: Explain the bug lifecycle.

A: The bug lifecycle is a series of states a bug goes through from discovery to resolution. The typical stages are: New → Assigned → Open → Fixed → Retest → Closed/Reopened.

Q: What is the difference between severity and priority?

A: Severity is the impact a bug has on the system (how bad it is), while priority is the urgency of fixing the bug (how soon it needs to be fixed).

Q: What happens when a bug is reopened?

A: A bug is reopened when a retest shows that the issue was not fixed properly by the developer and the bug still persists. It is then sent back to the developer for further attention.

Q: Why is tracking the bug lifecycle important?

A: It ensures that all defects are managed and resolved systematically. This process improves software quality, enhances team efficiency, and provides clear visibility into the status of each bug.
