Day 8 - Test Case Design Techniques

Introduction

To perform effective and comprehensive software testing, it is crucial to design high-quality test cases. Simply creating test cases randomly is inefficient and often leads to gaps in testing. Test case design techniques are systematic approaches that help QA professionals create a set of effective and efficient test cases, ensuring that all critical areas of the software are covered without redundant testing.

Types of Test Case Design Techniques

There are several techniques used to design test cases, each with its own purpose. These techniques can be broadly categorized into three types: Black-Box, White-Box, and Experience-Based. Here we focus on some of the most common ones.

Boundary Value Analysis (BVA)

This technique focuses on testing the values at the boundaries of valid input ranges. Bugs often occur at the edges, so testing these points is highly effective.

Example: For a text box accepting values from 1 to 100, BVA would test the following values:
  • Lower Boundary: 0 (invalid), 1 (valid)
  • Upper Boundary: 100 (valid), 101 (invalid)

Equivalence Partitioning (EP)

EP divides the input data into partitions (groups) that are expected to behave similarly. Instead of testing every single value, you only test one value from each partition, assuming it represents the entire group.

Example: An application requires the user's age. The age ranges are:
  • 1-18 (Child)
  • 19-60 (Adult)
  • 61-100 (Senior)
Using EP, you would select one test case from each valid partition (e.g., 10, 35, 75) and one from the invalid partitions (e.g., 0, 101).

Decision Table Testing

This technique is used for complex scenarios with multiple input conditions. It creates a table that shows all possible combinations of conditions and the corresponding actions, ensuring all logical rules are covered.

State Transition Testing

Used to test systems that have different states. Test cases are designed to test valid and invalid transitions between these states to ensure the system behaves correctly.

Use Case Testing

This technique derives test cases from "use cases," which describe how an actor (user) interacts with the system to achieve a specific goal. This approach is highly effective for testing real-world scenarios.

Error Guessing

This is an experience-based technique where the tester uses their knowledge and intuition to "guess" where defects might exist. It’s effective for finding defects that might be missed by formal techniques.

Comparison of Techniques

Technique Purpose Example
Boundary Value Analysis (BVA) Focuses on testing boundary values of input ranges. Test a password field accepting 6-12 characters with 5, 6, 12, and 13 characters.
Equivalence Partitioning (EP) Divides input data into partitions to reduce redundant test cases. Test a product review rating field (1-5 stars) with a single value from the valid partition (e.g., 3).
Decision Table Testing Tests complex business rules with multiple conditions. Test login functionality based on combinations of correct/incorrect username and password.

Interview Questions - Day 8

Q: What is the purpose of test case design techniques?

A: They are used to create efficient and effective test cases that cover various scenarios while minimizing redundancy. They help ensure comprehensive and systematic testing.

Q: Explain Boundary Value Analysis.

A: It's a black-box testing technique that focuses on testing input values at the extreme ends of valid and invalid ranges to find defects that often exist at these boundaries.

Q: What is Equivalence Partitioning?

A: It's a technique that divides the input data into groups or partitions. You only need to test one representative value from each partition, as all values within a partition are expected to produce the same outcome.

Q: How is a decision table useful in testing?

A: A decision table helps in systematically deriving test cases from a set of complex business rules. It maps all possible combinations of conditions to their corresponding actions, ensuring thorough test coverage of logical decisions.

Q: What is error guessing in testing?

A: Error guessing is a testing technique where a tester uses their experience, knowledge, and intuition about common software vulnerabilities to anticipate and "guess" where defects are most likely to occur, and then designs test cases specifically for those areas.
