Day 6 - Types of Testing

Introduction

Testing is a crucial phase in the software development lifecycle. It's the process of evaluating a software product to check if it meets specified requirements and to identify defects. Understanding different testing types helps ensure software quality and reliability.

Software testing is broadly categorized into two main types: Functional Testing and Non-Functional Testing.

Functional Testing

Functional Testing verifies that the software behaves according to the specified requirements. It focuses on 'what' the system does, checking all the functionalities and features of the application.

  • Unit Testing: Tests individual components or units of a software to validate that each unit works as designed.
    Example: Testing a single function that calculates a user's age based on their date of birth.
  • Integration Testing: Tests the interfaces between components to ensure that they work together correctly.
    Example: Testing the interaction between a login module and a dashboard module to ensure a user can access their account after successful login.
  • System Testing: Tests the complete and integrated software system to evaluate its compliance with the specified requirements.
    Example: Testing the entire e-commerce website, from product search to checkout, as a whole.
  • User Acceptance Testing (UAT): A final phase of testing where the end-users or clients verify if the system meets their needs.
    Example: The client tests the new CRM system to ensure it's ready for deployment in their business environment.

Non-Functional Testing

Non-Functional Testing validates software attributes like performance, usability, and security. It focuses on 'how' the system operates, checking its reliability, efficiency, and robustness.

  • Performance Testing: Evaluates how a system performs in terms of responsiveness and stability under a particular workload.
    Example: Checking the response time of a web page when 100 users are accessing it simultaneously.
  • Load Testing: A sub-type of performance testing that checks the system's behavior under expected load.
    Example: Simulating 5,000 concurrent users on a website to see if it remains stable.
  • Stress Testing: A sub-type of performance testing that pushes the system beyond its normal operating capacity to see how it handles extreme loads.
    Example: Simulating 10,000 concurrent users to find the breaking point of the system.
  • Security Testing: Checks the system for vulnerabilities and threats to protect data from potential attacks.
    Example: Verifying that a website is resistant to SQL injection or Cross-Site Scripting (XSS) attacks.
  • Usability Testing: Measures how easy and intuitive the software is to use.
    Example: Observing users interacting with a mobile app to see if they can easily navigate and complete tasks.

Other Types of Testing

Apart from the main categories, there are several other important testing types:

  • Smoke Testing: A quick, preliminary test to ensure the most basic and critical functionalities are working after a new build is deployed. It's a "go/no-go" decision.
  • Sanity Testing: A subset of regression testing that checks a specific, small part of the application after a minor change to ensure the change works and hasn't broken anything else in its immediate vicinity.
  • Regression Testing: Verifies that recent code changes (e.g., bug fixes, new features) have not broken or negatively affected existing functionalities.
  • Exploratory Testing: A hands-on, unscripted testing approach where the tester learns about the application while testing it. It's about exploring the software to find new issues.

Interview Questions - Day 6

Q: What is the difference between functional and non-functional testing?
A: Functional testing verifies the software functionality and what it does, while non-functional testing checks performance, usability, security, and how it performs.
Q: Name 3 types of functional testing.
A: Unit Testing, Integration Testing, System Testing.
Q: What is regression testing?
A: It verifies that recent code changes have not broken existing functionality.
Q: What is the difference between smoke and sanity testing?
A: Smoke testing is a quick check of basic functionality after a new build, whereas sanity testing is a focused check of specific functionality after a minor change or bug fix.
Q: Why is non-functional testing important?
A: It ensures software reliability, performance, security, and user satisfaction, which are crucial for the long-term success and adoption of the software.
